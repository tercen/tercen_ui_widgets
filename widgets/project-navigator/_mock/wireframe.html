LAYOUT RULES
Toolbar: fixed 48px height. Left-aligned buttons.
Upload (UP) = primary accent. Other buttons = ghost style.
Dividers separate button groups: [upload|download|refresh] | [team] | [filter+search].
Type filter is a labeled dropdown button, not an icon only.
Search = flex-grow text input, fills remaining toolbar width.
TREE RULES
Personal team: auto-expanded on load, listed first.
Other teams: collapsed on load.
Project contents: flat list (no virtual grouping).
Selected row: tinted bg + left accent border (3px).
Upload in progress: dashed border on drop target + inline progress bar.
Chevron + type icon + name + status badges, left to right.
NODE ICON KEY
P=Personal Team, T=Team, Pr=Project, Fo=Folder, F=File, Z=ZIP, D=Dataset, W=Workflow
STATUS BADGES
Project: version chip (v2.3), git-status dot (clean=green, modified=amber).
Workflow: status dot (idle=neutral, running=info, complete=success, error=error).
Team: member count in parentheses.

INTERACTION MODEL
Single-click branch: set focus + keep expanded
Click chevron: toggle expand/collapse only
Single-click leaf: set focus (emit focusChanged)
Double-click leaf: open viewer (emit openViewer)
Team single-click: expand/collapse only (no focus)

Level 1 (projects): 16px
